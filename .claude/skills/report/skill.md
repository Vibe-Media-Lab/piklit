# /report — 수정 리포트 생성

이번 세션에서 처리한 버그 수정 및 요청사항을 비개발자가 바로 이해할 수 있도록 정리한다.

## 실행 순서

### 1. 세션 변경 내역 수집
- 이번 대화에서 수정한 버그/요청사항 목록 (대화 컨텍스트에서 추출)
- `git log` 로 세션 중 생성된 커밋 확인

### 2. 리포트 작성
각 수정 건에 대해 아래 형식으로 정리:

```
## 수정 리포트 (YYYY-MM-DD)

| # | 문제 | 수정 내용 | 결과 |
|---|------|----------|------|
| 1 | 어떤 상황에서 무엇이 안 됐는지 (사용자 입장) | 어떻게 고쳤는지 (한 줄 요약) | 고친 후 어떻게 되는지 |
| 2 | ... | ... | ... |

### 상세

**1. [제목]**
> 문제: 사용자 관점에서 1줄 설명
> 해결: 무엇을 어떻게 바꿨는지 1줄 설명
> 커밋: `해시`
```

### 3. 출력 규칙
- 표(table)로 한눈에 요약 → 아래에 상세 설명
- 마크다운 형식으로 사용자에게 직접 출력 (별도 파일 생성 금지)
- 코드블록 없이 깔끔하게 출력

## 작성 톤
- **비개발자 기준**: 코드 파일명, 변수명, 기술 용어 사용 금지
- 사용자가 겪은 상황 → 어떻게 고쳤는지 → 이제 어떻게 되는지, 이 흐름만 전달
- "ObjectURL 초기화" (X) → "사진 정보가 임시 저장이라 페이지 이동하면 날아감" (O)
- "thinkingBudget 제거" (X) → "AI가 충분히 생각하지 않고 답을 내는 설정이었음" (O)
- 각 항목 1~2줄 이내로 간결하게
