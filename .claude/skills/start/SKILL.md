# /start — 세션 시작 자동화

세션 시작 시 프로젝트 상태를 빠르게 파악하고 작업 준비를 완료한다.

## 실행 순서

### 1. 프로젝트 상태 확인
다음 3가지를 **병렬로** 확인:
- `docs/task.md` 읽기 (진행 중 작업, 다음 목표)
- `docs/walkthrough.md` 읽기 (최근 변경 이력)
- `git status` + `git log --oneline -5` 실행

### 2. 좀비 프로세스 정리 & dev 서버 시작
```bash
# 기존 dev 서버 정리
lsof -ti:5173 | xargs kill -9 2>/dev/null || true
# dev 서버 백그라운드 시작
npm run dev
```

### 3. 상태 요약 보고
다음 형식으로 사용자에게 보고:

```
## 세션 시작 완료

**브랜치**: {현재 브랜치}
**마지막 커밋**: {최근 커밋 메시지}
**진행 중 작업**: {task.md에서 확인한 현재 작업}
**다음 목표**: {task.md에서 확인한 다음 목표}
**dev 서버**: http://localhost:5173

무엇을 하시겠습니까?
```

## 주의사항
- 파일 구조 전체 탐색 금지 — task.md와 walkthrough.md만 확인
- 불필요한 파일 읽기 금지
- 보고는 간결하게 (10줄 이내)
