# /commit — 빠른 커밋

빌드 확인 → 변경 사항 분석 → 커밋 메시지 제안 → 사용자 승인 → 커밋.

## 실행 순서

### 1. 빌드 확인
```bash
npm run build
```
- 실패 시: 즉시 수정 후 재빌드
- 성공 시: 다음 단계

### 2. 변경 사항 확인
```bash
git status
git diff --staged
git diff
```
- 변경 파일 목록을 사용자에게 간결히 보여줌

### 3. 커밋 메시지 제안
변경 내용을 분석하여 컨벤셔널 커밋 메시지 제안:
- `feat:` 새 기능
- `fix:` 버그 수정
- `refactor:` 리팩토링
- `style:` UI/CSS 변경
- `docs:` 문서 변경
- 한국어, 1~2문장, "왜"에 초점

### 4. 사용자 승인 후 커밋
```bash
git add <변경 파일들>    # 특정 파일만 (git add -A 지양)
git commit -m "제안된 메시지"
git status               # 커밋 결과 확인
```

### 5. 결과 보고
```
커밋 완료: {해시} — {메시지}
변경 파일: {N}개
```

## 주의사항
- `.env`, credentials 등 민감 파일 커밋 금지
- push는 하지 않음 (사용자가 명시적으로 요청 시에만)
- 빌드 실패 상태에서 커밋하지 않음
- 사용자가 메시지를 수정하면 수정된 메시지로 커밋
