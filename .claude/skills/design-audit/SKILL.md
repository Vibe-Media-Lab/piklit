# /design-audit -- 디자인 감사

JSX에서 사용하는 CSS 클래스와 CSS 파일에 정의된 클래스를 비교하여 불일치를 검출한다.

## 실행 순서

### 1. 대상 결정
사용자가 파일명을 지정하면 해당 파일만, 아니면 최근 변경된 JSX/CSS 파일을 대상으로 한다.

기본 대상 페어:
- `LandingPage.jsx` <-> `landing.css`
- `EditorPage.jsx` <-> `components.css`, `tiptap.css`
- `HistoryPage.jsx` <-> `history.css`
- `PostListPage.jsx` <-> `components.css`

### 2. 클래스 추출
**JSX에서**: `className="..."` 패턴으로 사용된 모든 CSS 클래스 추출
- 정적 className, 템플릿 리터럴, 조건부 className 모두 포함
- 자식 컴포넌트에서 사용하는 클래스도 추적

**CSS에서**: 선택자로 정의된 모든 클래스 추출

### 3. 비교 분석
다음 3가지를 보고:

**A. 누락 CSS (HIGH)**: JSX에서 쓰지만 CSS에 정의 없음
- 스타일이 적용되지 않아 레이아웃이 깨질 수 있음

**B. 미사용 CSS (LOW)**: CSS에 정의되어 있지만 JSX에서 안 씀
- 즉시 문제는 아니지만 번들 크기 낭비

**C. 반응형 누락 (MEDIUM)**: 768px/480px 미디어 쿼리에서 빠진 클래스
- 모바일에서 레이아웃이 깨질 수 있음

### 4. 결과 보고
```
## 디자인 감사 결과

### 누락 CSS (JSX에서 사용, CSS 미정의)
| 클래스명 | 사용 파일:줄번호 | 심각도 |
|---------|----------------|--------|

### 미사용 CSS (CSS 정의, JSX 미사용)
| 클래스명 | CSS 파일:줄번호 |
|---------|----------------|

### 반응형 누락
| 클래스명 | 768px | 480px |
|---------|-------|-------|

**요약**: 누락 {N}개 / 미사용 {N}개 / 반응형 누락 {N}개
```

## 주의사항
- 읽기 전용 — 코드 수정하지 않음
- `variables.css`의 CSS 변수는 검사 대상에서 제외
- lucide-react 등 외부 라이브러리 클래스는 제외
- 공통 클래스(landing-section-badge 등)는 부모 선택자 컨텍스트 고려
