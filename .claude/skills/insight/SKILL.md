# /insight — 세션 인사이트 보고

현재 세션의 작업 통계와 패턴을 분석하여 한국어로 보고한다.

## 실행 순서

### 1. 데이터 수집
다음 명령을 병렬 실행:
```bash
# 최근 커밋 이력 (이번 세션)
git log --oneline --since="today" --stat

# 변경 파일 통계
git diff --stat HEAD~$(git log --oneline --since="today" | wc -l | tr -d ' ')..HEAD 2>/dev/null || git diff --stat

# 전체 프로젝트 커밋 통계 (최근 30일)
git log --oneline --since="30 days ago" | wc -l

# 전체 프로젝트 파일 수
find src/ -name "*.jsx" -o -name "*.js" -o -name "*.css" | wc -l
```

### 2. 이번 세션 분석
대화 컨텍스트에서 추출:
- 이번 세션에서 수정한 파일 수, 추가/삭제 라인
- 사용한 주요 도구 (Read, Edit, Bash, Grep, Agent 등)
- 수행한 작업 유형 (버그 수정, 기능 추가, 설정 변경 등)
- 발생한 이슈와 해결 방법

### 3. 아래 형식으로 출력

```
기간: {세션 시작일} ~ {오늘} ({N}일)

📊 전체 통계
- {메시지 수} 메시지 / {커밋 수} 커밋
- +{추가 라인} / -{삭제 라인} 라인 변경
- {수정 파일 수} 파일 수정
- 소스 파일: {총 소스 파일 수}개

🎯 주요 작업 영역
1. {영역 1} ({파일 수 또는 작업 수})
2. {영역 2}
3. {영역 3}

✅ 잘하고 있는 점
- {패턴 1}
- {패턴 2}

⚠️ 개선 필요 사항
1. {이슈 1}
2. {이슈 2}

💡 추천 액션
1. {제안 1}
2. {제안 2}

🧠 미래 워크플로우 제안
- {제안 1}
- {제안 2}

📈 사용 패턴
- 주요 도구: {도구 사용 패턴}
- 작업 유형: {가장 많은 작업 유형}
- 세션 특징: {특이사항}
```

## 주의사항
- 모든 수치는 실제 git 로그·대화 컨텍스트 기반 (추측 금지)
- 수집 불가능한 수치는 "미확인"으로 표시
- 간결하게 — 각 섹션 3항목 이내
