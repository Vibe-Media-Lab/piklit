# /pre-commit-qa — 커밋 전 품질 점검

커밋 전에 코드 품질을 자동 점검하고, 문제 발견 시 수정 후 커밋 진행.

## 실행 순서

### 1. 빌드 확인
```bash
npm run build
```
- 실패 시 즉시 수정

### 2. 린트 확인
```bash
npm run lint
```
- 에러: 즉시 수정
- 경고: 목록 보고 (사용자 판단)

### 3. 코드 품질 체크
변경된 파일에서 다음 항목 검사:
- `console.log` / `console.warn` / `console.error` 잔존 (디버그용)
- 하드코딩된 API 키 또는 URL
- 미사용 import

### 4. 변경 요약
```
## 커밋 전 점검 결과

- 빌드: ✅ / ❌
- 린트: ✅ 에러 0 / ⚠️ 경고 N개
- console.log: ✅ 없음 / ⚠️ N개 발견
- 변경 파일: N개 (파일 목록)
```

### 5. 결과에 따라
- 모두 통과 → "커밋할까요?"
- 문제 발견 → 자동 수정 후 재검증 → "커밋할까요?"

## 주의사항
- 점검 범위: 이번 세션에서 변경된 파일만
- 수정 범위: 점검에서 발견된 문제만 (추가 리팩토링 금지)
