# /qa -- QA 요청 메시지 생성

메인 에이전트에서 실행. 변경 파일을 수집하고 QA 탭에 붙여넣을 메시지를 자동 생성한다.

## 실행 순서

### 1. 변경 파일 수집
```bash
git diff --name-only HEAD
git diff --staged --name-only
```
- 변경 없으면: "변경 사항이 없습니다" 출력 후 종료

### 2. 변경 유형 분류
변경 파일을 아래 카테고리로 자동 분류:
- **CSS/스타일**: `.css` 파일
- **컴포넌트**: `.jsx` 파일
- **서비스/유틸**: `.js` 파일 (services/, utils/)
- **설정**: `vite.config.js`, `package.json` 등
- **문서**: `.md` 파일

### 3. 체크포인트 자동 생성
변경 유형에 따라 체크포인트를 자동 선택:

- CSS 변경 시: "JSX에서 사용하는 클래스가 모두 CSS에 정의되어 있는지 확인"
- 컴포넌트 변경 시: "빌드 오류 없는지 확인", "import 경로 정상인지 확인"
- openai.js 변경 시: "API 호출 패턴이 CLAUDE.md 규칙을 따르는지 확인"
- EditorContext 변경 시: "의존 컴포넌트에 영향 없는지 확인"
- landing.css/LandingPage.jsx 변경 시: "반응형 3단계(1024/768/480px) 확인"

### 4. QA 메시지 출력
아래 형식으로 출력 (사용자가 복사해서 QA 탭에 붙여넣기):

```
## QA 요청

아래 확인을 코드 수정 없이 분석하고 보고만 해줘.

### 변경 파일
- {파일 목록}

### 체크포인트
- [ ] npm run build 정상
- [ ] npm run lint 경고 0개
- [ ] {자동 생성된 체크포인트들}

### 변경 요약
{변경 내용 1~2줄 요약}
```

## 주의사항
- 메시지 생성만 함 (QA 에이전트를 직접 실행하지 않음)
- 사용자가 복사-붙여넣기 할 수 있도록 코드블록으로 출력
