# /fix-batch — 버그 일괄 수정

사용자가 보고한 버그 목록을 순서대로 분석·수정·검증한다.

## 실행 순서

### 1. 버그 목록 파악
- 사용자 메시지에서 버그 항목 추출 (번호 매기기)
- 각 버그별 관련 파일 특정

### 2. 순차 수정
각 버그에 대해:
1. 원인 분석 (파일:줄번호 특정)
2. 수정 적용
3. `npm run build` 검증
4. 빌드 실패 시 즉시 수정, 다음 버그로 넘어가지 않음

### 3. 수정 리포트 출력
```
## 수정 완료 (N건)

| # | 버그 | 원인 | 수정 파일 | 상태 |
|---|------|------|----------|------|
| 1 | ... | ... | ... | ✅ |
| 2 | ... | ... | ... | ✅ |
```

### 4. 커밋 확인
"커밋할까요?" 로 사용자에게 확인

## 주의사항
- 버그 수정 범위를 벗어난 리팩토링 금지
- 수정 건별로 빌드 확인 필수
- 한 버그 수정이 다른 버그에 영향 줄 수 있으니 순서 고려
