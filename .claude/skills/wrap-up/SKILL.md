# /wrap-up — 세션 종료 자동화

세션 종료 시 빌드 확인, 문서 업데이트, 커밋을 순차적으로 처리한다.

## 실행 순서

### 1. 빌드 확인
```bash
npm run build
```
- 빌드 실패 시: 즉시 수정 후 재빌드 (사용자에게 알림)
- 빌드 성공 시: 다음 단계로 진행

### 2. 문서 업데이트
변경 사항을 기반으로 다음 파일 업데이트:

**`docs/task.md`**:
- 완료된 작업 체크 표시
- 새로 발견된 TODO 추가
- "다음 세션" 섹션 업데이트

**`docs/walkthrough.md`**:
- 이번 세션에서 변경한 파일 목록
- 주요 변경 내용 요약 (1~3줄)

### 3. Git 커밋 (사용자 승인 후)
```
git add -A
git status  # 사용자에게 변경 파일 보여주기
# 커밋 메시지 제안 후 사용자 승인 대기
git commit -m "제안된 메시지"
```

### 4. 세션 종료 보고
```
## 세션 종료 완료

**변경 파일**: {N}개
**커밋**: {커밋 해시} — {커밋 메시지}
**다음 세션 목표**: {task.md에서 확인}

수고하셨습니다!
```

## 주의사항
- 커밋 전 반드시 사용자 승인 받을 것
- push는 하지 않음 (사용자가 명시적으로 요청 시에만)
- 빌드 실패 상태에서 커밋하지 않음
