# 피클잇 명령어 치트시트

## 스킬 (슬래시 명령어)

| 명령어 | 설명 | 언제 쓰나 |
|--------|------|----------|
| `/start` | 세션 시작 — 상태 파악 + dev 서버 시작 | 작업 시작할 때 |
| `/qa` | 변경 파일 수집 + QA 붙여넣기 메시지 생성 | 코드 수정 후 QA 요청할 때 |
| `/design-audit` | JSX vs CSS 클래스 불일치 검출 | CSS/UI 변경 후 검증할 때 |
| `/deploy-check` | 배포 전 점검 (빌드+환경변수+보안) | 배포 직전 |
| `/commit` | 빌드 확인 + 커밋 메시지 제안 + 커밋 | 코드 변경 저장할 때 |
| `/wrap-up` | 문서 업데이트 + 빌드 + 커밋 + 종료 보고 | 작업 끝날 때 |

## 커맨드

| 명령어 | 설명 | 언제 쓰나 |
|--------|------|----------|
| `/team` | 에이전트팀 셋업 안내 (QA/디자인QA/프롬프트 탭) | 세션 시작 후 팀 구성할 때 |

## 에이전트팀 (별도 터미널 탭)

| 탭 이름 | 역할 | 핵심 규칙 |
|---------|------|----------|
| **메인** | 개발 총괄, 코드 수정 | 유일하게 코드를 수정하는 에이전트 |
| **QA** | 빌드/린트 검증, 코드 리뷰 | 코드 수정 금지, 보고만 |
| **디자인QA** | CSS 감사, 반응형 체크, 디자인 시스템 준수 | 코드 수정 금지, 보고만 |
| **프롬프트** | AI 프롬프트 분석/개선안 | 코드 수정 금지, 제안만 |

## 자동 훅

| 트리거 | 동작 | 설명 |
|--------|------|------|
| 파일 수정 후 (Edit/Write) | `npm run build` | 빌드 오류 자동 감지 |

## 세션 흐름 요약

```
/start              세션 시작, 현황 파악
  |
/team               에이전트팀 셋업 (QA/디자인QA/프롬프트)
  |
(작업)              메인에서 코드 수정
  |
/qa                 QA 요청 메시지 생성 → QA 탭에 붙여넣기
  |
/design-audit       CSS 변경 시 디자인 감사
  |
/commit             중간 저장
  |
/deploy-check       배포 전 점검
  |
/wrap-up            세션 마무리
```

## QA 에이전트에 붙여넣는 메시지

```
빌드 + 린트 확인해줘.
변경된 파일: (메인에서 알려준 파일 목록)
```

## 프롬프트 에이전트에 붙여넣는 메시지

```
openai.js의 [메서드명] 프롬프트 분석해줘.
개선안이 있으면 변경 전/후로 보여줘.
```
