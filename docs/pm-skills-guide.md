# 피클잇 PM Skills 활용 가이드

> PM Skills 플러그인(pm-skills)을 피클잇 제품 개발에 단계별로 활용하는 방법

---

## Phase 1 — MVP 런칭 준비 (지금 당장)

### 1. GTM 전략 수립

| 커맨드 | 활용 방법 |
|--------|----------|
| `/plan-launch` | "네이버 블로거 대상 사진→블로그 자동 생성 서비스" 런칭 전략 한 번에 수립. 비치헤드 세그먼트 → ICP → GTM 전략까지 |
| `/value-proposition` | "사진만 올리면 SEO 최적화 블로그 글 완성"의 가치 제안을 6파트 JTBD 템플릿으로 명확화 |
| `/competitive-analysis` | 뤼튼, 블로그지기, ChatGPT 등 경쟁 서비스 5개 분석 → 차별화 포인트 도출 |

**왜 중요**: MVP를 만들었어도 "누구에게 먼저, 어떤 메시지로" 알릴지 명확하지 않으면 런칭이 흐지부지됩니다.

### 2. 비치헤드 세그먼트 정의

| 커맨드 | 활용 방법 |
|--------|----------|
| `/research-users` | 네이버 블로거 유형별 페르소나 생성 (맛집 리뷰어, 여행 블로거, 제품 리뷰어, 일상 블로거 등) |
| `beachhead-segment` | 4가지 기준(고통점, 지불의지, 시장 점유 가능성, 추천 잠재력)으로 첫 타겟 확정 |

**예시**: "주 3회 이상 포스팅하지만 글쓰기에 매번 2시간+ 소요되는 맛집/카페 블로거"가 비치헤드일 수 있음

### 3. 포지셔닝 & 마케팅

| 커맨드 | 활용 방법 |
|--------|----------|
| `/market-product` | 마케팅 아이디어 5개 + 포지셔닝 5가지 한 번에 생성 |
| `positioning-ideas` | "Pickle + Lit = 사진을 글로 절이다" 콘셉트를 경쟁사 대비 차별화된 5가지 포지셔닝으로 발전 |

## Phase 2 준비 — 결과물 품질 강화, 대시보드

### 4. 제품 발견 & 우선순위

| 커맨드 | 활용 방법 |
|--------|----------|
| `/discover` | 품질 개선 방향 탐색: 아이디어 발산 → 가정 식별 → 실험 설계까지 풀 사이클 |
| `/interview prep` | 초기 사용자 인터뷰 스크립트 생성 — "블로그 글 작성 시 가장 고통스러운 부분은?" |
| `/interview summarize` | 인터뷰 후 녹취록 → JTBD/통증점/만족도 구조화 |
| `/triage-requests` | 사용자 피드백/기능 요청 분류 및 우선순위화 |
| `/analyze-feedback` | 초기 사용자 피드백 대량 분석 → 감정점수 + 주제 추출 |

### 5. 메트릭 설계

| 커맨드 | 활용 방법 |
|--------|----------|
| `/north-star` | 피클잇의 북스타 메트릭 정의 (예: "주간 발행된 블로그 글 수" 또는 "AI 생성 글의 네이버 검색 노출 수") |
| `/setup-metrics` | 북스타 + 입력 메트릭 + 건강 메트릭 + 경고 임계값 설계 |

### 6. 제품 실행

| 커맨드 | 활용 방법 |
|--------|----------|
| `/write-prd` | Phase 2 기능별 PRD 작성 (대시보드, 클라우드 저장 등) |
| `/write-stories` | PRD → 유저스토리/잡스토리로 분해 |
| `/test-scenarios` | 각 기능의 테스트 시나리오 자동 생성 |
| `/sprint` | 스프린트 계획 → 회고 → 릴리스 노트 사이클 |

## Phase 3 준비 — BYOK + 구독 결제

### 7. 수익화 전략

| 커맨드 | 활용 방법 |
|--------|----------|
| `/pricing` | 프리미엄/구독/BYOK 가격 모델 설계 + 경쟁 벤치마크 + 지불의지 추정 |
| `/business-model` | Lean Canvas 또는 Startup Canvas로 비즈니스 모델 검증 |
| `monetization-strategy` | 3-5개 수익 모델 비교 (프리미엄, 구독, 사용량 기반 등) |

### 8. 성장 전략

| 커맨드 | 활용 방법 |
|--------|----------|
| `/growth-strategy` | 성장 루프 설계 — 피클잇은 "사용자가 만든 블로그 글이 곧 마케팅"이므로 **UGC 루프**가 핵심 |
| `/battlecard` | "피클잇 vs 뤼튼" 같은 영업용 경쟁 카드 |

|--------|----------|
| `/strategy` | 9섹션 제품 전략 캔버스 — 비전/세그먼트/가치제안/트레이드오프/메트릭/성장/역량/Can't Won't |

## 추천 실행 우선순위 (Top 5)

지금 피클잇 상황에서 **가장 임팩트가 큰 순서**:

| 순위 | 커맨드 | 목적 |
|------|--------|------|
| 1 | `/plan-launch` | MVP 런칭 전 GTM 전략 한 번에 세우기 |
| 2 | `/value-proposition` | "왜 피클잇이어야 하는가" 명확화 |
| 3 | `/north-star` | 성공을 측정할 핵심 지표 정의 |
| 4 | `/research-users` | 타겟 블로거 페르소나 & 세그먼트 확정 |
| 5 | `/pricing` | Phase 3 전에 가격 모델 미리 설계 |
