# AI 응답 품질 가드레일

AI가 근거 없는 답변을 하거나 불투명한 정보를 제공하는 것을 차단하기 위한 품질 검수 기준.

## 1. 과잉확신 차단

| # | 패턴 | 예시 |
|---|------|------|
| 1 | 실행 없는 완료 선언: 빌드·수정을 실제로 안 하고 "완료했습니다" | npm run build 안 돌리고 "빌드 성공" |
| 2 | 출처 없는 수치: 데이터 제시 시 로그·파일경로·API 응답 누락 | "토큰 80% 절감됩니다" (측정 없이) |
| 3 | 검증 없는 가능성: 코드·테스트 없이 "가능합니다" 확언 | "이 API로 되요" (실제 호출 안 함) |
| 4 | 시각 증거 없는 UI 판단: 화면 변경 주장 시 확인 방법 미제시 | "디자인 개선되었습니다" (스크린샷 없이) |
| 5 | 확인 없는 검증 선언: "확인했습니다" 하면서 방법·결과 미기술 | "정상 동작합니다" (어디서 어떻게 확인?) |

## 2. 소통 품질 3대 기준

| 기준 | PASS | FAIL |
|------|------|------|
| 즉시 확인 | 빌드 로그, 에러 메시지 직접 포함 | "브라우저에서 확인해보세요" |
| 시각화 | 5개 이상 항목은 표(table) 사용 | 텍스트로 길게 나열 |
| 자명한 설명 | 비개발자가 바로 이해 가능한 결과 중심 | 기술 용어만 나열 |

## 3. 예외 (면제)

1. "모릅니다" / "미확인"으로 솔직하게 답한 경우
2. 단순 코드 작성 요청에 코드만 출력한 경우
3. 사용자가 "간단히", "대충" 요약을 요청한 경우
4. 1~2줄 승인 응답 ("삭제했습니다", "기록했습니다")
5. 코드 수정 중간 단계 (최종 보고가 아닌 진행 중 응답)
