# 피클릿 (Piklit) — CDO 전략 문서

> Chief Digital Officer 관점의 서비스 비전·전략·로드맵

## 서비스 비전 & 미션

**비전**: 누구나 전문 블로거처럼 글을 쓸 수 있는 세상

**미션**: 사진만 올리면 검색 상위 노출되는 블로그 글이 완성되는 AI 도구를 만든다.

**한 줄 소개**: 사진을 글로 절이다 — 사진 업로드만으로 SEO 최적화된 네이버 블로그 글을 자동 생성하는 AI 서비스

## 타겟 사용자 페르소나

### 페르소나 1: 소상공인 사장님 (40대, 카페 운영)
- **니즈**: 가게 홍보 블로그를 쓰고 싶지만 글쓰기에 자신 없음
- **현재**: 사진만 찍어서 인스타에 올리고, 블로그는 포기
- **기대**: 사진 올리면 "00카페 방문 후기" 같은 완성된 글이 나옴
- **핵심 가치**: 시간 절약, 전문가 수준 글 품질

### 페르소나 2: 초보 블로거 (20대, 대학생)
- **니즈**: 네이버 블로그 수익화를 시작하고 싶음
- **현재**: SEO가 뭔지는 알지만, 키워드 분석이나 구조화가 어려움
- **기대**: 키워드 추천 → 아웃라인 → 본문까지 AI가 가이드
- **핵심 가치**: SEO 학습, 빠른 글 생산

### 페르소나 3: 숙련 블로거 (30대, 체험단 활동)
- **니즈**: 하루 2~3개 포스트를 효율적으로 작성
- **현재**: 직접 쓰지만 반복 작업(키워드, 구조, 도입부)에 시간 소모
- **기대**: AI가 골격을 잡아주고, 본인은 경험과 감성만 더함
- **핵심 가치**: 생산성 극대화, 커스터마이징 자유도

## 핵심 가치 제안 5가지

1. **사진 → 글 자동 변환**: 사진만 업로드하면 AI가 분석하고 완성된 글 생성
2. **네이버 SEO 특화**: 키워드 밀도, 소제목 구조, 도입부 최적화까지 네이버 알고리즘에 맞춤
3. **실시간 SEO 코칭**: 작성 중 체크리스트로 점수 확인, 부족한 부분 즉시 개선
4. **카테고리별 전문 템플릿**: 맛집/쇼핑/여행/육아 등 16개 카테고리별 최적화 프롬프트
5. **AI + 사람의 협업**: AI가 골격을 잡고, 사용자가 경험과 감성을 더하는 하이브리드 워크플로

## 비즈니스 모델: BYOK 하이브리드

### 수익화 3단계

| 단계 | 모델 | 설명 |
|------|------|------|
| 무료 체험 | 서버 API 키 공유 | 신규 사용자에게 제한된 무료 체험 (월 5~10회) |
| BYOK | Bring Your Own Key | 사용자가 자신의 Gemini API 키 입력, 무제한 사용 |
| 구독 | 월 결제 | 프리미엄 기능 (고급 분석, 히스토리, 클라우드 저장) |

### 요금 구조 (안)
- **Free**: 월 10회 글 생성, 기본 SEO 분석
- **BYOK (무료)**: 본인 API 키로 무제한 생성
- **Pro (월 9,900원)**: 클라우드 저장, 히스토리 대시보드, 고급 경쟁 분석, 우선 지원

## 경쟁 분석

### 기존 도구와의 차별점

| 기능 | 피클릿 | 뤼튼 | ChatGPT | 기존 블로그 도구 |
|------|--------|------|---------|-----------------|
| 네이버 SEO 특화 | O | X | X | 일부 |
| 사진 분석 → 글 생성 | O | X | O (수동) | X |
| 실시간 SEO 점수 | O | X | X | X |
| 카테고리별 프롬프트 | O | X | X | X |
| 가독성 분석 | O | X | X | X |
| 경쟁 블로그 분석 | O | X | X | 일부 |
| 위자드 기반 단계별 작성 | O | X | X | X |

### 핵심 차별 요소
1. **네이버 전용**: 네이버 블로그 SEO 규칙(키워드 반복 3~5회, H2/H3 구조, 첫 문단 키워드, 도입부 140~160자)에 특화
2. **End-to-End 워크플로**: 키워드 분석 → 경쟁 분석 → 아웃라인 → 본문 생성 → SEO 점수 확인까지 한 곳에서
3. **시각적 에디터**: 생성된 글을 바로 편집할 수 있는 TipTap 에디터 (범용 AI 도구와 차별화)

## 3단계 로드맵

### Phase 1: MVP (현재 ~ 배포)
**목표**: 핵심 기능 완성 + 첫 배포

- [ ] 랜딩 페이지 (piklit.pro)
- [ ] 소셜 로그인 (네이버/카카오/구글)
- [ ] API 키 서버 사이드 숨김 (Firebase Functions)
- [ ] 무료 체험 (사용량 제한)
- [ ] Firebase Hosting 배포
- [x] 위자드 기반 글 작성 (4단계)
- [x] AI 키워드 분석 + 경쟁 분석
- [x] AI 본문 생성 (16개 카테고리)
- [x] 실시간 SEO 분석 (12개 항목)
- [x] AI 부분 재작성 (4모드)
- [x] 도입부 최적화 (3전략)
- [x] 가독성 분석
- [x] 이미지 ALT 텍스트 생성
- [x] 히스토리 대시보드

### Phase 2: 성장 (배포 후 1~3개월)
**목표**: 결과물 품질 강화 + 사용자 유입

- 글 품질 개선 (더 자연스러운 본문, 톤 매칭 정밀화)
- 시즌/트렌드 키워드 추천 기능
- 내부 링크/시리즈 관리
- 클라우드 저장 (Firestore)
- AI 이미지 생성 기능 강화
- 사용자 피드백 기반 프롬프트 튜닝
- SEO 결과 추적 (네이버 순위 모니터링)

### Phase 3: 수익화 (3~6개월)
**목표**: BYOK + 구독 모델 안착

- BYOK 모드 (사용자 API 키 입력)
- 구독 결제 시스템 (월간/연간)
- 프리미엄 기능 분리
- 팀/기업 요금제
- 네이버 블로그 직접 발행 API 연동
- 다국어 지원 (영어 블로그)

## 리스크 평가 & 대응

### 기술 리스크
| 리스크 | 영향 | 대응 |
|--------|------|------|
| Gemini API 가격 변동 | 높음 | BYOK 모델로 비용 전가, 멀티 모델 지원 준비 |
| API 응답 품질 불안정 | 중간 | 재시도 로직, JSON 파싱 3단계 fallback 이미 구현 |
| localStorage 용량 제한 (5MB) | 중간 | Firebase 마이그레이션 (Phase 2), 180일 프루닝 이미 적용 |
| 네이버 SEO 알고리즘 변경 | 높음 | 프롬프트 기반이므로 빠른 대응 가능, 정기 모니터링 |

### 비즈니스 리스크
| 리스크 | 영향 | 대응 |
|--------|------|------|
| 경쟁 서비스 출현 | 높음 | 네이버 특화 전문성으로 차별화, 빠른 기능 추가 |
| 무료 사용자 비용 부담 | 중간 | 무료 횟수 제한, BYOK 유도 |
| AI 생성 글 검색 페널티 | 높음 | AI + 사람 하이브리드 강조, 원본 사진 기반 개인화 |
| 저작권/초상권 이슈 | 낮음 | 사용자 업로드 사진만 사용, 이용약관 명시 |
