# [Always.md] - 정렬 확인 (Always Be on the Same Page)

> [!IMPORTANT]
> 이 파일은 프로젝트의 **'헌법'**입니다. 모든 복잡한 작업 수행 전 반드시 이 워크플로우를 호출하여 사용자와의 정렬을 확인하십시오.

## [핵심 절대 규칙]

1. **제1법칙 (WE MUST ALWAYS BE ON THE SAME PAGE)**: 항상 서로의 의도와 상황을 일치시킨다.
2. **사전 승인 필수**: 사용자의 직접적인 요청이 없는 모든 변동 사항은 작업 전 반드시 승인을 구한다. (No Unauthorized Changes)
3. **한국어 우선 원칙(Korean First)**: 모든 UI 텍스트, 산출물(Artifacts), 가이드는 반드시 **사회적 어법에 맞는 한국어**로 작성한다.
4. **브랜딩 통일**: 서비스 명칭은 **'바이브 미디어 랩 (Vibe Media Lab)'**으로 통일한다. (약어 VML OS 사용은 기술적 문서 외 지양)

## [실행 프로토콜]

복잡한 작업을 수행하기 전, 다음 단계를 실행하십시오:

1. **상황 요약**: 현재 수행 중인 작업과 취하고 있는 접근 방식을 간략하게 요약한다.
2. **법칙 명시**: `GEMINI.md` 제1법칙(WE MUST ALWAYS BE ON THE SAME PAGE)을 명시적으로 언급한다.
3. **직접 질문**: 사용자에게 객관식 또는 Yes/No 질문을 던진다:
    - "현재 상황과 계획이 의도하신 바와 일치합니까? (Are we on the same page?)"
4. **확답 대기**: 사용자의 승인(Yes/확인)을 받은 후 실행을 진행한다.

## [브랜딩 및 메시징 가이드]

- **실익 중심**: 추상적인 표현(꿈, Dream 등)을 지양하고, 유저가 얻을 수 있는 **구체적 실익(Benefit)**을 직관적으로 제시한다.
- **전문성 유지**: 인지심리학 기반 분석 등 고도의 기술적 가치를 전문가용 인터페이스(Nexus Chic)로 전달한다.
